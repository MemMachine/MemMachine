This page documents the current Python data structures used by MemMachine for
episodic memory.

Primary source modules:
- `memmachine.common.episode_store.episode_model`
- `memmachine.common.api.spec` (client-safe API DTOs)

## ID Alias

### `EpisodeIdT`

```python
EpisodeIdT = str
```

Unique string identifier for an episode.

## `ContentType` (Enum)

```python
class ContentType(Enum)
```

Specifies the data format in an episode `content` field.

| Member | Value |
| --- | --- |
| `STRING` | `"string"` |

## `EpisodeEntry`

```python
class EpisodeEntry(BaseModel)
```

Write payload used when creating a new episode.

| Field | Type | Required |
| --- | --- | --- |
| `content` | `str` | Yes |
| `producer_id` | `str` | Yes |
| `producer_role` | `str` | Yes |
| `produced_for_id` | `str \| None` | No |
| `episode_type` | `EpisodeType \| None` | No |
| `metadata` | `dict[str, JsonValue] \| None` | No |
| `created_at` | `AwareDatetime \| None` | No |

## `EpisodeResponse`

```python
class EpisodeResponse(EpisodeEntry)
```

Episode DTO returned by search APIs.

| Field | Type | Notes |
| --- | --- | --- |
| `uid` | `EpisodeIdT` | Episode identifier |
| `score` | `float \| None` | Relevance score (search responses) |

## `Episode`

```python
class Episode(BaseModel)
```

Stored episode model with persistence metadata.

| Field | Type | Required |
| --- | --- | --- |
| `uid` | `EpisodeIdT` | Yes |
| `content` | `str` | Yes |
| `session_key` | `str` | Yes |
| `created_at` | `AwareDatetime` | Yes |
| `producer_id` | `str` | Yes |
| `producer_role` | `str` | Yes |
| `produced_for_id` | `str \| None` | No |
| `sequence_num` | `int` | No (default `0`) |
| `episode_type` | `EpisodeType` | No (default `EpisodeType.MESSAGE`) |
| `content_type` | `ContentType` | No (default `ContentType.STRING`) |
| `filterable_metadata` | `dict[str, PropertyValue] \| None` | No |
| `metadata` | `dict[str, JsonValue] \| None` | No |

## Session-Scoped Access (`SessionData`)

`MemMachine` operations that read/write memory use a session-scoped context
object (protocol) with:
- `org_id: str`
- `project_id: str`
- `session_key: str`

This replaces older docs that described a separate `MemoryContext` dataclass.
